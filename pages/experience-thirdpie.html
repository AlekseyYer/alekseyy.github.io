Overview

At Third Pie Studios, I was contracted to help bring Ages of Cataria, an indie village simulation game made in Unreal Engine 5, to a publishable Early Access state. My responsibilities included identifying and fixing bugs, implementing UI, programming in Blueprints, and collaborating with a small, fully remote team of programmers, writers, 3D artists, and a music composer. Through this opportunity, I was fortunate to make several key contributions that expanded my technical skill set.

Seasons Update

Impact Run-time seasonal transitions across thousands of actors with no measurable performance cost

One notable feature I took initiative on building was the Seasons Update. Prior to this work, I had limited experience with Unreal Engine's material and shader systems; however, through an iterative process, I built a performant system that changes in-game seasons. To make this work, I utilized Unreal's Material Parameter Collection (MPC) tool, as well as Dynamic Material Instances. Additionally, I used the Material Graph to swap textures via a scalar parameter in an MPC. I created several new Niagara VFX as well, including wind, snow, leaves, and emissive particles. Together, these systems allow thousands of Blueprint actors, landscapes, foliage, and static meshes to transition between seasons at runtime with no measurable performance impact.

Music System

Impact Data-driven music system supporting randomized day/night tracks with support for seasonal variations.

I also took the lead on implementing a music system that plays randomized day and night tracks based on the current time-of-day cycle and active season. While selecting a random track for just the current cycle was straightforward, I had to refactor the system to account for season-specific music. To do this, I used a struct that contains a cycle integer and an array of associated sound assets. I then utilized a Primary Data Asset Blueprint to map a season enum to a corresponding array of the aforementioned struct. Seperate data assets for each season were created, allowing season-specific tracks to be assigned to certain cycles. Because of this data driven approach, the system is scalable and allows for music variations without code changes.

Portrait Capture System

Impact Eliminated inconsistent portrait renders caused by occlusion and lighting with a queue system.

Another significant contribution involved refactoring and optimizing an existing villager portrait capture system. The original approach relied on per-actor scene capture cameras, which did not scale well and frequently produced inconsistent results due to world occlusion and uncontrolled lighting conditions. I redesigned the system around a centralized photobooth actor equipped with a single scene capture camera, controlled lightning, and a mannequin. Portrait requests are processed through a queue, allowing villagers to be handled sequentially. Each villager's mesh and materials are applied to the mannequin, the capture is performed, and the system advances to the next request. As a result, the system became more performant and ensured consistent visual output.

Settings And Save/Load System

Impact Full settings menu with graphics, resolution, frame rate, and key mapping, including save/load support.

In addition, I built a settings menu and the save/load functionality for it. This included display options such as resolution, window mode, graphics presets, frame rate limits, as well as key mappings. Audio controls were implemented using Unreal's Sound Classes and Mixes to manage music, sound effects, and ambient audio.
