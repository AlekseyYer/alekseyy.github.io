Overview

Project Pathfinder is a custom implementation of the A* pathfinding algorithm in Unreal Engine 5. Coded in C++, it features an interactive hexagonal grid system and movement calculations. The purpose of this project was to strengthen my data structures and algorithms skills. Furthermore, I created detailed documentation explaining the code and role of each class.

Hexagonal Grid System

Impact Generates a dynamic hex grid with movement weights and neighbor references using a hash map for fast lookups.

The first challenge in this personal project was to create a hexagonal grid that could efficiently render large numbers of hex tiles. To do this, I used an instanced static mesh component. Each grid tile is placed in offset rows to create the hexagonal layout, and is mapped onto a Grid Node object using a hash map. These nodes support the pathfinding algorithm by storing information such as:

  • its coordinates on the grid
  • the node's world position
  • the node's movement cost
  • whether or not the node is an obstacle
  • and references to the node's neighbors.

Once all tiles are placed, the grid builds neighbor relationshup by checking adjacent coordinates.

A* Pathfinding

Impact Implented A* pathfinding with obstacle avoidance, weighted movement cost, and path reconstruction.

The pathfinding system uses a TArray as the open set to store candidate nodes that still need to be evaluated. A TSet is used as the closed set to track nodes that have already been processed and should not be revisited. Each node stores the traversal cost, the heuristic cost, and a parent reference to the previous node. During execution, the algorithm repeatedly selects the lowest-cost node from the open set , evaluates its neighbors, and updates their costs when a cheaper path is found. Once the goal node is reached, the final path is reconstructed by following parent points back to the start.

Interactivity

Impact User can randomize the grid, designate start/goal nodes, toggle obstacles, and change the size of the grid.

A custom player controller handles all grid interaction, allowing players to designate start and goal nodes with left-clicks and toggle obstacles, using right-clicks. The controller performs raycasts against the instanced mesh to identify the node instance. Any change to the grid triggers a path recalculation. Through a simple UI menu, the player can also randomize the grid and change its size.
